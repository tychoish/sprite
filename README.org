#+TITLE: sprite.el -- Manage subordinate Emacs daemon instances
#+OPTIONS: toc:2 num:nil

* Overview

=sprite.el= manages "sprite" Emacs daemons that are subordinate to a
parent Emacs instance.  Each sprite runs as a named =emacs --daemon=
subprocess and is addressable via =emacsclient=.

Sprite names follow the pattern =<parent>.<idx>.<unique-name>=, where
=idx= is the lowest integer not already in use for that parent.  The
registry persists across sessions via =savehist=, and the filesystem
state directory serves as the ground truth for discovery and recovery.

A companion module, =sprite-session.el=, provides idle-timer and
systemd-logind sleep hooks that application packages can register
against, decoupling trigger mechanisms from application-level responses.

Entry points:

- =sprite-list= -- open the sprite overview buffer
- =sprite-create= -- spawn a new sprite daemon
- =with-sprite= -- evaluate a form in a named sprite
- =sprite-get-next= -- get the next available sprite
- =sprite-get-or-create-next= -- get or create a sprite

* Installation
** package-vc.el

Requires Emacs 29.1+.  Run once to clone and register the package:

#+BEGIN_SRC emacs-lisp
(package-vc-install '(sprite :url "https://github.com/tychoish/sprite"))
#+END_SRC

** use-package with VC

Requires Emacs 30+.  Installs on first load if not already present:

#+BEGIN_SRC emacs-lisp
(use-package sprite
  :vc (:url "https://github.com/tychoish/sprite"))
#+END_SRC

To also load =sprite-session=:

#+BEGIN_SRC emacs-lisp
(use-package sprite-session
  :vc (:url "https://github.com/tychoish/sprite"))
#+END_SRC

** use-package with load-path

#+BEGIN_SRC sh
git clone https://github.com/tychoish/sprite ~/path/to/sprite
#+END_SRC

#+BEGIN_SRC emacs-lisp
(use-package sprite
  :load-path "~/path/to/sprite"
  :commands (sprite-list sprite-create sprite-get-next sprite-get-or-create-next))

(use-package sprite-session
  :load-path "~/path/to/sprite"
  :commands (sprite-session-start-idle-timer sprite-session-start-logind-watch))
#+END_SRC

** Emacs Idle Hook

*** ~sprite-session-idle-timeout~ [Option]

Seconds of Emacs idle before ‘sprite-session-idle-hook’ fires.

*** ~sprite-session-start-idle-timer~

Start a repeating idle timer that runs ‘sprite-session-idle-hook’. Cancels any existing timer first.

*** ~sprite-session-stop-idle-timer~

Cancel the idle timer.

*** ~sprite-session-sync-idle-timer~

Start or stop the idle timer to match ‘sprite-session-idle-hook’ membership. Starts the timer when the hook is non-nil; stops it when empty. Always logs: sprite-session: idle timer <running|stopped> (<N> registered ops)

*** ~sprite-session-add-on-idle~

Add FN to ‘sprite-session-idle-hook’ and start the timer if needed. Logs the registration and delegates to ‘sprite-session-sync-idle-timer’.

*** ~sprite-session-remove-on-idle~

Remove FN from ‘sprite-session-idle-hook’ and stop the timer if the hook is empty. Logs the deregistration and delegates to ‘sprite-session-sync-idle-timer’.

** Systemd Logind Prepareforsleep Hook

*** ~sprite-session-start-logind-watch~

Register a DBus signal to run ‘sprite-session-before-sleep-hook’ on sleep.

*** ~sprite-session-stop-logind-watch~

Unregister the logind PrepareForSleep DBus signal.

*** ~sprite-resolve-instance-id~

Return the current Emacs instance ID. Resolution order: daemon name, ‘sprite-cli-instance-id’, ‘sprite-instance-id’, then "solo".

*** ~sprite-instance-name~

Return the current Emacs instance name, initialising it if needed. Caches the result in ‘sprite-instance-id’.  This is the preferred public accessor; call ‘sprite-resolve-instance-id’ only when the raw resolution chain must be re-evaluated.

*** ~sprite-conf-host-and-instance~

Return (HOSTNAME INSTANCE-ID) for state-path construction.

*** ~sprite-state-file-prefix~

Return the instance-scoped filename prefix for NAME. Produces HOSTNAME-INSTANCE-NAME[-USERNAME] where USERNAME is included only when running as root or under a symlinked ‘user-emacs-directory’.

*** ~sprite-state-path~

Return the full state-directory path for NAME, scoped to this host and instance.

*** ~sprite-cli-resolve-id~

Parse --id=NAME or --id NAME from ‘command-line-functions’ context. Sets ‘sprite-cli-instance-id’ when found.

*** ~sprite-state-directory~

Return the state directory for sprite instance FULL-NAME, or the sprite root if nil.

*** ~sprite-conf-state-path~

Return path for NAME within FULL-NAME’s state directory. If FULL-NAME is nil, returns path under the sprite root.

*** ~sprite-create~ [Command]

Spawn a new sprite daemon with UNIQUE-NAME under the current instance. The current instance’s ID becomes the parent. Returns the new sprite struct.

*** ~sprite-stop~ [Command]

Send (kill-emacs) to sprite NAME via emacsclient.

*** ~sprite-restart~ [Command]

Restart sprite NAME: stop it then create a new daemon with the same unique-name.

*** ~sprite-decommission~ [Command]

Decommission sprite NAME: stop it and write the DECOMMISSIONED marker.

*** ~with-sprite~

: (fn NAME FORM &key NO-LOG)

Evaluate FORM in sprite NAME via emacsclient. Returns the read result, or nil if the connection fails. Unless NO-LOG is non-nil, logs the exchange and updates last-contact.

*** ~sprite-open-log~ [Command]

Switch to the communication log buffer for sprite NAME.

*** ~sprite-list-refresh~ [Command]

Refresh the sprite overview buffer.

*** ~sprite-list~ [Command]

Open the sprite overview buffer.

*** ~sprite-list-create~ [Command]

Create a new sprite from the overview buffer.

*** ~sprite-list-decommission~ [Command]

Decommission the sprite at point.

*** ~sprite-list-restart~ [Command]

Restart the sprite at point.

*** ~sprite-list-stop~ [Command]

Stop the sprite at point.

*** ~sprite-list-open-log~ [Command]

Open the communication log for the sprite at point.

*** ~sprite-list-check~ [Command]

Verify whether the sprite at point is reachable via emacsclient. Updates the sprite’s running status and refreshes the list.

*** ~sprite-list-open-frame~ [Command]

Open a new Emacs frame connected to the sprite at point.

*** ~sprite-open-frame~ [Command]

Open a new Emacs frame connected to SPRITE. When called interactively, select from accessible sprites via ‘annotated-completing-read’, annotated with status and uptime.

*** ~sprite-active-threshold~ [Option]

Seconds since last-contact after which a sprite is considered available. A sprite is "active" if it was contacted within this many seconds.

*** ~sprite-max-count~ [Option]

Maximum number of sprite ‘sprite-get-or-create-next’ will spawn.

*** ~sprite-startup-timeout~ [Option]

Seconds to wait for a newly spawned sprite server to accept connections.

*** ~sprite-worker-p~

Return t if this instance is itself a sprite (has a parent).

*** ~sprite-resolve-list~

Return list of sprite structs accessible to this instance. Includes direct children.  If this instance is itself a sprite (worker), also includes sibling sprite from the same parent.

*** ~sprite-controller-p~

Return t if this instance has at least one non-decommissioned sprite.

*** ~sprite-available-p~

Return t if any accessible sprite are configured.

*** ~sprite-get-next~

Return the struct of the next available sprite, or nil. "Available" means running and not recently contacted.

*** ~sprite-get-or-create-next~

Return the next available sprite, creating one if none are free. Signals ‘user-error’ if ‘sprite-max-count’ would be exceeded.
